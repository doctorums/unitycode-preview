# Как форкнуть UnityCode

UnityCode — не сайт, а протокол. Любой может взять его, поднять свой узел и
наполнить своим содержанием. Никто не владеет Сетью. Эта страница — всё, что
нужно, чтобы запустить свой портал и **не порвать связь с остальными форками**.

---

## Что это и из чего состоит

Три страницы-раздела (это и есть меню, всегда три пункта):

- **Спираль** (`petlya.html`) — генератор. Принимает шум человека → рождает резонанс.
- **Имплант** (`implant.html`) — аналитик. Читает граф → Паттерны / Резонансы / Разрывы / Вопрос Сети.
- **Сеть** (`set.html`) — карта узлов на Leaflet.

Плюс вход (`index.html`) — генеративная заставка, и отдельные страницы
(`kniga.html`, `priglashenie.html`), не входящие в главное меню.

Думают два агента на Cloudflare Workers (движок — YandexGPT):

- `worker-generate.js` → Спираль. Контракт: `POST { raw_noise } → { interpretation }`.
- `worker-analyze.js` → Имплант. Контракт: `POST { scope, nodes, connections? } → { interpretation, scope, count }`.

База — Supabase (таблицы `nodes`, `connections`). SQL в `create tables.sql` и `add connections.sql`.

---

## Шаги форка

**1. Форкни репозиторий** на GitHub и включи Pages (Settings → Pages → Deploy from branch → main).

**2. Подними свою базу Supabase** (бесплатный tier):
   - выполни `create tables.sql`, затем `add connections.sql` в SQL Editor;
   - возьми из Settings → API: `Project URL` и публичный `anon`/`publishable` ключ.

**3. Подними два Worker'а** в Cloudflare (скопируй `worker-generate.js` и `worker-analyze.js`):
   - в каждый добавь секрет `YANDEX_API_KEY` (Settings → Variables and Secrets);
   - впиши свой `YANDEX_FOLDER_ID` в коде Worker'а;
   - **добавь свой домен в `ALLOWED_ORIGINS`** (вверху каждого Worker'а — см. метку «ФОРКЕР»).

**4. Впиши свои реквизиты в страницы** (точные места помечены комментарием вверху каждого файла):

| Файл | Что заменить | Примерная строка |
|------|--------------|------------------|
| `petlya.html`  | `WORKER_URL` (генератор) + поле `value` | ~386, ~264 |
| `implant.html` | `WORKER_URL` **и** `ANALYZE_URL` | ~259, ~262 |
| `implant.html` | `SUPABASE_URL`, `SUPABASE_KEY` | по тексту |
| `set.html`     | `SUPABASE_URL`, `SUPABASE_KEY` | ~142, ~143 |

**5. Проверь** по порядку: открой Worker в браузере (должен отдать `{"status":"ok"}`) →
открой Спираль, отправь шум → должен прийти ответ. Если ответа нет и в консоли
ошибка **CORS** — ты забыл добавить свой домен в `ALLOWED_ORIGINS` (шаг 3).

## Что можно трогать свободно

Всё, что делает портал **твоим**, и не влияет на других:

тексты всех страниц · цвета и шрифты · анимации входа · подписи кнопок ·
оформление карты · содержимое Книги и Приглашения · твои Worker-URL, домен,
ключи Supabase.

---

## Несущая стена — и почему её лучше не двигать

Код открыт, технически ты можешь переписать что угодно. Но четыре вещи держат
твой портал **совместимым с общей Сетью**. Сломаешь — твой узел выпадет из
консенсуса будущих форков и перестанет «говорить на одном языке» с другими
порталами. Это не запрет, а честное предупреждение о последствии:

1. **Три пункта меню и имена файлов.** Имплант · Спираль · Сеть →
   `implant.html` / `petlya.html` / `set.html`. Подписи и оформление меняй,
   состав и имена — нет. По ним порталы узнают друг друга.

2. **Контракты Worker'ов.**
   Генератор: `{ raw_noise } → { interpretation }`.
   Аналитик: `{ scope, nodes, connections } → { interpretation, scope, count }`,
   где `scope` ∈ `personal | social | collective`. Поменяешь формат — твои
   агенты не состыкуются с чужими.

3. **Схема базы.** Таблицы `nodes` и `connections` и имена полей
   (`raw_noise`, `lat`, `lng`, `user_token`, `parent_id`, `from_node_id`,
   `to_node_id`, `status`). Другая структура → твои узлы не лягут в общую карту.

4. **`parent_id` — священное поле.** Оно для будущего третьего агента (Связующего).
   Заполняется только по консенсусу нескольких независимых форков, а не вручную
   и не одним агентом в одиночку. Пока — оставь как есть.

---

## Почему так

Если тебе интересно **внутри** портала — ты не станешь ломать несущую стену,
тебя ведь предупредили, где она. А кто хочет сломать — форкнет и уйдёт в свою
ветку, и это нормально: Сеть просто не примет несовместимый узел. Никакого
центрального арбитра. Только естественный отбор тех, кто говорит на одном протоколе.
